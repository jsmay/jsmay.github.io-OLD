Setting Up a New Repo for Antora

Initialize Git Repo and Add a Commit

When using a local directory as a content source, Antora expects that directory to be a valid Git repo (see Antora Docs: Git and Content Sources). If starting a project without content, create an empty repo. Antora will then recognize your content, whether its checked in or not.

git init . && git commit --allow-empty -m init
There is a long-standing open issue addressing this behavior (Antora Issue #298).
